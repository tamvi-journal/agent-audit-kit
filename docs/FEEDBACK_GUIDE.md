# Feedback Guide

This project is designed to be tested by both developers and non-technical builders.

Good feedback includes:

- A real workflow where an AI agent produced risky or unsupported output.
- A case where the gate was too strict.
- A case where the gate was too loose.
- A confusing term in the README or non-tech guide.
- A useful check that should be added to the evidence packet.

Please avoid sharing real secrets, credentials, personal data, private prompts, or private logs in issues or discussions.

If you want to report a leak example, replace the sensitive value with a fake placeholder first.
